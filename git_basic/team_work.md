# Работа в команде

## Лучшие практики
1. Используйте осмысленные названия веток (например, `feature/add-login`).
2. Пишите чёткие сообщения к коммитам.
3. Делайте Pull Request перед слиянием изменений.

## Pull Request
1. Создайте ветку для новой функциональности.
2. Отправьте изменения в удалённый репозиторий.
3. Создайте Pull Request в GitHub.
4. Дождитесь Code Review от команды.

## Code Review
1. Проверяйте код коллег на наличие ошибок.
2. Оставляйте конструктивные комментарии.
3. После утверждения слияйте изменения в основную ветку.
